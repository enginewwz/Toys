环境是 wsl，如果是 windows 可能需要改个头文件，还有中间有个 system("clear") 得改

使用以下方式储存数据：
1. 64 位宽无符号整数储存棋盘（0代表未落子， 1代表已落子），使用位操作修改棋盘值
2. 长度为 64 的 128 位静态栈，使用宏定义定义栈操作，使用循环而非递归方式，减少程序调用带来的性能损耗，另一方面也容易打印每一步的状态。总大小 1K ，保证能够留在L1缓存（好吧实际上太充裕了）。当然另一方面这也导致每次回溯时（不过貌似根本不回溯）需要重新计算邻接点最少的位置。
3. 正式搜索前提前计算每个位置的周围八个（或更少）的可能位置，程序运行时能直接拿到掩码

使用 Warnsdorff 方式剪枝搜索，具体介绍可以参考 wiki。实际上就是每次选择可活动范围最小的位置走
